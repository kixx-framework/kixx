Callouts

A boxed notice that flags a message without derailing the page.

A callout lifts one message out of the surrounding copy — a rule of thumb, a caution, an error — onto a sunken surface inside a hairline box with a heavier left marker, and an optional ASCII mark. It’s the calm way to raise your voice: present enough to catch the eye, quiet enough not to shout.

It’s the first real consumer of the status colors. The tone rides the left marker and the mark — graphical elements, which clear WCAG’s 3:1 non-text threshold — while the message stays --color-ink. That’s deliberate: it’s why the warning tone resolves to a deep amber on light surfaces rather than the brand yellow, which is too light to read as a thin mark on paper.

When to use: to surface a single, scannable message tied to its context. For a transient confirmation that auto-dismisses, reach for a toast instead; for page-blocking decisions, a dialog. A callout is a note on the page, not an interruption of it — and a page stacked with callouts has none.

Anatomy

Four parts; the mark is optional, the tone is a modifier. The default callout is a left marker, a body, and a surface.

  1. Left marker — a 3px left border (--callout-accent) on an otherwise hairline box; the tone modifier recolors it.
  2. Mark — optional ASCII glyph ([!], [i], [x], *), inherits --callout-accent; reinforces the tone so meaning survives without color (.callout__icon).
  3. Body — the message (.callout__body): an optional label plus copy, always in --color-ink.
  4. Surface — the recessed fill (--color-surface-sunken) that lifts the note off the page.

Variants

A base class plus three tone modifiers. Each tone resolves from a semantic token — no raw color anywhere.

  • .callout — the neutral emphasis note; the marker is --color-ink. Reach for it for a plain “rule of thumb.”
  • .callout--info — the link blue (--color-ink-link); calm, non-urgent information.
  • .callout--warning — amber on light, yellow on dark (--color-status-warning); proceed with caution.
  • .callout--error — red (--color-status-danger); something failed or is blocked.
*

Rule of thumb

The default tone. One note per idea — if everything is highlighted, nothing is.

Scheduled maintenance runs Sunday 02:00–03:00 UTC. No action needed.

This API key has read-write access. Rotate it if it’s ever been shared.

[x]

Couldn’t save

Usage

Wrap the message in .callout__body so the optional mark and the copy lay out side by side. The mark is decorative — the tone word in the body carries the meaning.

<div class="callout callout--warning">
<span class="callout__icon" aria-hidden="true">[!]</span>
<div class="callout__body">
<p class="type-label">Warning</p>
<p>Proceed-with-caution message goes here.</p>
</div>
</div>

<!-- Neutral note: no modifier, no mark needed -->
<div class="callout">
<div class="callout__body"><p>A plain rule of thumb.</p></div>
</div>

<!-- An error raised after an action — announce it to assistive tech -->
<div class="callout callout--error" role="alert">…</div>

Accessibility

  • Never lean on the marker color alone — pair the tone with the ASCII mark and a word (“Warning”, “Couldn’t save”) so it reads in greyscale and for color-blind users.
  • The mark is decorative once the text says the same thing; give it aria-hidden="true" so it isn’t announced.
  • For a callout that appears in response to an action (a save error, say), add role="alert" — or role="status" for non-urgent info — so screen readers announce it. A note present on load needs neither.
  • Keep the message in --color-ink; it sits on --color-surface-sunken and clears AA. Don’t recolor body text to the tone — the status hues are tuned for marks (3:1), not for text (4.5:1).

Guidelines

Do

  • Match the tone to the stakes: info for FYIs, warning before risk, error for failure.
  • Keep it to one idea, and put the gist in the first few words.
  • Pair every tone with its ASCII mark and a label word, not color alone.
  • Let warning and error ride the marker and mark; leave the body text in --color-ink.

Don’t

  • Stack callouts down the page — competing alarms read as none at all.
  • Paint the body text or the whole surface in the tone color, especially yellow.
  • Use error tone for ordinary emphasis; reserve red for things that are actually broken.
  • Reach past the tone modifiers to a raw --palette-* or an hsl() literal.
