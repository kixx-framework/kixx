Typography

How to effectively use typography in the Kixx Admin Panel theme.

Typography is the primary design element in this theme. Hierarchy is carried by scale and weight — not color, underlines, or decoration — so the system stays small, predictable, and honest about being an admin panel rather than a landing page.

Two registers, plus labels and code

Every piece of text belongs to one of four groups:

  • Expressive (DM Sans) — the structural headings h1–h4. Heavy weights and tight line-height, used sparingly for impact.
  • Functional (Helvetica Neue) — reading and UI text, from .type-body-lg down to .type-caption.
  • Label / overline (mono, uppercase) — the theme's signature for small functional text. The buttons and the sidebar title already use it.
  • Code (mono) — inline tokens, values, and snippets like --text-h1.

Why every size is a fixed rem

Marketing sites use fluid clamp() headings so a hero looks huge on desktop and still fits a phone. An admin panel has no heroes: headings are structural labels inside a capped content column, and the window stays roughly one size all day. So we use fixed rem steps instead:

  • Predictable density. A section title is the same size every time, so dense screens and data tables stay stable instead of breathing with the viewport.
  • Zoom & accessibility. Fixed rem scales cleanly with browser zoom; vw-based fluid type partially ignores it, working against WCAG 1.4.4 (Resize Text).
  • Simplicity. If a heading is ever too large on mobile, add one discrete breakpoint step — never continuous scaling.

Rule of thumb

Line-height is role-dependent, never one global value. Big headings run tight (1.15), reading copy opens to 1.55 for breathability, and dense UI text sits near 1.4 to save vertical space.

Guidelines

Do

  • Reach for semantic elements first — h1–h4 and p render correctly with no classes.
  • Signal importance with size and weight, not color, italics, or underlines.
  • Let paragraphs breathe: .type-body-lg at 1.55 for anything people actually read.
  • Keep a single h1 per screen as the page title.

Don't

  • Pick a heading level for its size — choose the level for structure, then restyle with a .type-* class.
  • Invent new font sizes; every size lives in a --text-* token.
  • Reintroduce fluid clamp() type — fixed rem only, with a breakpoint step if needed.
  • Drop reading copy below 1.5 line-height.
