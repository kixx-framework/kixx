Buttons

The action control — a mono, uppercase label inside a square, bracketed box.

Buttons trigger actions. The label is set in the mono label register — uppercase and tracked — wrapped in brackets inside a square, 1px box, so an action reads like a terminal target you can type at: [ save ].

When to use: a <button> performs an action (save, delete, open a dialog); a link navigates somewhere. If pressing it changes the URL, it’s a link — you can style it .button, but keep the <a> element. Keep a single primary action per view.

Anatomy

Every button is the same square box; emphasis comes from the fill, never from size or a brand hue.

  1. Label — a verb in the mono label register: uppercase, tracked.
  2. Brackets — [ and ] framing the label, added in CSS so it reads as a spec-sheet control.
  3. Box — a 1px solid edge (--button-border-width) on square corners — no radius.
  4. Fill — the page surface for the default button; solid ink for the one primary action.
  5. Target — a 2.5rem minimum height for a comfortable, consistent target.
  6. Focus ring — a 2px --color-focus-outline outline, offset 2px (keyboard only).

Variants

Three variants only. Emphasis comes from fill; the one use of color is the destructive signal.

.buttonDefault — outline, inverts on hover
.button--primaryThe one primary action — solid ink fill
.button--dangerDestructive — color as signal

States

Hover, focus (press Tab), and click the live buttons to see each state. States are inversions and border shifts — no shadow, no motion.

These are real controls — hover, tab to them, and press.

Default
Resting. The default button sits on the page surface with an ink box; primary carries the solid ink fill.
Hover
Inversion. The default button fills with ink; the primary flips to an outline; danger fills red — a simple flip, no transition.
Focus
A 2px --color-focus-outline ring, offset 2px. Shows for keyboard focus (:focus-visible), not mouse clicks.
Disabled
Box and label go to --color-ink-muted on the page surface, so a disabled button reads as inactive. Hover is suppressed.

Usage

Match the element to the job: <button> for actions, <a> for navigation.

<!-- Primary action — one per view -->
<button class="button button--primary" type="button">Save changes</button>

<!-- Default / neutral action -->
<button class="button" type="button">Cancel</button>

<!-- Destructive action -->
<button class="button button--danger" type="button">Delete</button>

<!-- Navigation that looks like a button: keep the <a> -->
<a class="button" href="/settings">Open settings</a>

<!-- Disabled -->
<button class="button button--primary" type="button" disabled>Saving…</button>

Accessibility

  • Use <button> for actions and <a> for navigation — keyboards and screen readers treat them differently; don’t swap them.
  • Always set type on a <button> (button or submit) so it can’t submit a form by accident.
  • The visible label is the accessible name; keep it a clear verb. The brackets are added in CSS, so they never reach the accessibility tree.
  • Never remove the focus ring — :focus-visible shows it for keyboard users only.
  • Buttons hold a 2.5rem minimum height for a comfortable target; keep interactive controls at least this size.
  • When a control is disabled but users still need to know it exists, prefer aria-disabled="true" (it stays focusable and announced) over the disabled attribute.

Guidelines

Do

  • Keep one primary button per view; make everything else the default outline.
  • Lead the label with a verb — “Save changes,” “Delete,” “Invite.”
  • Let the fill carry emphasis and trust the fixed sizes.
  • Reserve .button--danger for actions that destroy something.

Don’t

  • Stack several primary buttons competing for the same attention.
  • Use a button to navigate — that’s a link’s job.
  • Remove the focus ring or drop below the 2.5rem target.
  • Add a shadow, a radius, or a hover transition — states are inversions, not motion.
