Buttons

The action control — a mono, uppercase label in a rounded, pill-shaped target.

Buttons trigger actions. The label is set in the mono label register — uppercase and tracked — inside a fully-rounded, 2px-bordered pill, so an action reads as a deliberate, approachable thing to press.

When to use: a <button> performs an action (save, delete, open a dialog); a link navigates somewhere. If pressing it changes the URL, it's a link — you can style it .button, but keep the <a> element. Keep a single primary action per view.

Anatomy

Every button is the same shape; only the fill changes between variants.

  1. Label — a verb in the mono label register: uppercase, +0.04em tracking.
  2. Border — a 2px solid edge (--button-border-width) on fully-rounded pill corners (--radius-pill).
  3. Fill — the page surface for plain buttons, a brand color for emphasis.
  4. Target — a 2.5rem minimum height for a comfortable, consistent target.
  5. Focus ring — a 3px --color-focus-outline outline, offset 2px (keyboard only).

Variants

One plain button plus three brand fills. Emphasis comes from fill, not size.

.buttonPlain / neutral — inverts to ink on hover
.button--primaryThe one primary action
.button--secondarySecondary emphasis
.button--tertiaryTertiary emphasis

States

Hover, focus (press Tab), and click the live buttons to see each state.

These are real controls — hover, tab to them, and press.

Default
Resting. Plain sits on the page surface with an ink border; brand variants carry their fill.
Hover
Every variant inverts to a solid ink fill — the brand fill drops away so all buttons share one hover treatment.
Focus
A 3px --color-focus-outline ring, offset 2px. Shows for keyboard focus (:focus-visible), not mouse clicks.
Active
Nudges down 1px on press — a tactile, theme-agnostic cue.
Disabled
Brand fill and border are replaced with --color-surface-sunken and --color-ink-muted — all variants go grey so none reads as a muted brand action. Hover and press are suppressed.

Usage

Match the element to the job: <button> for actions, <a> for navigation.

<!-- Primary action — one per view -->
<button class="button button--primary" type="button">Save changes</button>

<!-- Secondary / neutral action -->
<button class="button" type="button">Cancel</button>

<!-- Navigation that looks like a button: keep the <a> -->
<a class="button button--secondary" href="/settings">Open settings</a>

<!-- Disabled -->
<button class="button button--primary" type="button" disabled>Saving…</button>

Accessibility

  • Use <button> for actions and <a> for navigation — keyboards and screen readers treat them differently; don't swap them.
  • Always set type on a <button> (button or submit) so it can't submit a form by accident.
  • The visible label is the accessible name; keep it a clear verb. An icon-only button needs an aria-label.
  • Never remove the focus ring — :focus-visible shows it for keyboard users only, so it doesn't clutter mouse clicks.
  • Buttons hold a 2.5rem minimum height for a comfortable, consistent target; keep interactive controls at least this size.
  • When a control is disabled but users still need to know it exists, prefer aria-disabled="true" (it stays focusable and announced) over the disabled attribute.

Guidelines

Do

  • Keep one primary button per view; make everything else plain or secondary.
  • Lead the label with a verb — “Save changes,” “Delete,” “Invite.”
  • Let the variant carry emphasis and trust the fixed sizes.
  • Reach for .button (plain) for low-emphasis and cancel actions.

Don't

  • Stack several primary buttons competing for the same attention.
  • Use a button to navigate — that's a link's job.
  • Remove the focus ring or drop below the 2.5rem target.
  • Invent new fills; the four variants map to the brand palette.
