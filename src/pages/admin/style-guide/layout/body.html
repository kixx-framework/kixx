Layout

The composable primitives that arrange everything else in the Kixx Admin Panel theme.

Layout is handled by six small, single-purpose primitives in the Every Layout tradition. Each is one class, tuned by its own scoped custom properties — so you arrange a page by nesting primitives, not by writing a new flex or grid rule every time.

Every primitive defaults to a design token, so the untouched class already does the right thing. To adjust one axis, set its custom property on the element (or any ancestor):

  • .flow — vertical stack; rhythm between siblings via --flow-space.
  • .cluster — horizontal group that wraps; --cluster-space, --cluster-align, --cluster-justify.
  • .grid-auto — auto-fit grid; tracks never below --grid-min, gap via --grid-space.
  • .switcher — row that flips to a column under --switcher-threshold.
  • .with-sidebar — fixed-ish sidebar (--sidebar-width) beside fluid content (--sidebar-content-min).
  • .center — caps a column to --center-max and centers it.

Rule of thumb

This page is built from them: the header bar is a .cluster, and the whole app shell is a .with-sidebar. Reach for a primitive before a bespoke rule — and never nest .flow directly inside .flow (the inner one re-scopes --flow-space and steals its own top margin).

Guidelines

Do

  • Compose layout by nesting primitives — a .cluster inside a .flow inside a .with-sidebar.
  • Adjust one axis by setting its custom property, not by writing a new rule.
  • Let .grid-auto and .switcher reflow on their own; trust the content threshold.
  • Keep tuning values on the design tokens (--space-*, --measure-prose).

Don't

  • Reach for a bespoke display: flex / grid rule before checking the six primitives.
  • Nest .flow directly inside .flow — use a plain owl for the inner stack.
  • Add media-query breakpoints where .grid-auto or .switcher already adapt.
  • Hard-code pixel gaps; the spacing ladder is the contract.
