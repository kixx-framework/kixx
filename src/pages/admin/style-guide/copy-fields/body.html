Copy Fields

A labeled, read-only value paired with a one-click copy button.

A copy field is the display primitive for any value someone needs to take away verbatim — a one-time secret, a shareable link, an identifier. It reuses the text field’s input treatment so the value looks like a real form control, then adds a copy button beside it and a small status line below to confirm the copy.

It is deliberately progressive. Without JavaScript the value is a plain readonly input — selectable and copyable by hand. With JavaScript, the delegated data-js-behavior="copy-field" script adds click-to-select on the value, a one-click clipboard write on the button, and status feedback that clears itself after a moment.

When to use: a copy field is for a value the user will paste somewhere else — an API token, an invite URL, an account ID, a recovery code. Reach for it whenever getting the value out cleanly matters more than editing it. For a value the user types, use a text field. For a value that is only displayed, plain text is enough.

Anatomy

A label above, then a row pairing the read-only value with its copy button, and a status line below that speaks only after a copy.

  1. Label — mono, uppercase, muted; tied to the value input with a matching for/id pair, exactly like a field label.
  2. Value — a readonly .field__input so it carries the field’s border, fill, font, and dashed read-only border; it scrolls long values natively and selects on click.
  3. Button — a square control matched to the input height; it copies the value and swaps its text icon to a success mark when the clipboard write succeeds.
  4. Status — an aria-live line that stays empty until a copy, then announces Copied. or the manual-copy fallback. Its height is reserved so the row never shifts.

States

Click the value to select it, or press the button to copy. This is a live control — the status line and icon respond in real time.

This is a real control — press the button to copy, or click into the value.

Idle
Value in an input with a dashed readonly border; the button sits quiet beside it. The status line is empty.
Value hover / focus
Clicking the value selects the whole string; focusing it shows the standard field focus ring.
Button hover
The button inverts to a solid --color-ink fill — a clear, contained affordance without moving anything.
Button focus
A 2px --color-focus-outline ring on :focus-visible for keyboard users.
Copied
On success the icon becomes a success mark and the status reads Copied., then both reset after two seconds.
No clipboard
Where the Clipboard API is unavailable or denied, the value is selected and the status falls back to Press Ctrl+C or Cmd+C to copy.

Usage

Wire the button to the value with data-copy-target pointing at the input’s id. The behavior binds itself; there is no per-page script to add.

<div class="copy-field">
<label class="copy-field__label" for="invite-url">Invite link</label>
<div class="copy-field__row">
<input class="field__input copy-field__value" id="invite-url"
type="text" value="https://kixx.dev/..." readonly>
<button class="copy-field__button" type="button"
data-js-behavior="copy-field" data-copy-target="invite-url"
aria-label="Copy invite link">
<span class="copy-field__icon" aria-hidden="true">copy</span>
</button>
</div>
<p class="copy-field__status" role="status" aria-live="polite"></p>
</div>

Accessibility

  • Connect the <label> to the value input with a matching for/id pair — the value is a real, labeled control, not decoration.
  • Give the button an aria-label that names what it copies, such as Copy invite link, because its only visible content is an icon.
  • Mark the copy icon aria-hidden="true" so it is not announced separately from the button’s label.
  • Keep the status line as an aria-live="polite" region so copy feedback is announced without stealing focus; its reserved height also prevents a layout shift.
  • Leave the value readonly rather than disabled — a disabled input is skipped by assistive technology and cannot be selected or copied by hand.

Guidelines

Do

  • Use it for any value the user needs to paste elsewhere — tokens, links, IDs, recovery codes.
  • Pair a one-time secret with a callout that says it is shown only once and cannot be retrieved again.
  • Label the value by what it is, so a screen reader announces Invite link, not just text field.
  • Trust the progressive fallback: the plain readonly input is still selectable when scripting is off.

Don’t

  • Use disabled on the value — it can no longer be selected or copied by hand.
  • Drop the aria-label and leave the button an unlabeled icon.
  • Reuse the same id for two copy fields on one page; each value needs a unique target.
  • Use a copy field for a value the user is meant to edit — that is a text field.
