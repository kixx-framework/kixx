Cards

The bordered surface primitive the rest of the UI sits on.

A card is a flat, hairline-bordered panel — the structural building block for dashboards, settings groups, and list modules. Square corners, a 1px rule, a flat fill: a spec-sheet box, not a decorative surface. Depth comes from the rule, never a shadow.

Because it’s built entirely from semantic tokens — --color-rule-strong for the border, --color-bg for the fill, and --color-ink / --color-ink-muted for the accent marker — a card is correct in both light and dark with no extra work. It’s a small, concrete demonstration of the color contract.

When to use: group related content into a distinct surface. When you only need to space things out, reach for .flow or a layout primitive instead — a card is a surface, not a spacer.

Anatomy

Four parts, with the accent marker optional. The default card is just a surface, a hairline box, and padding.

Notification preferences

A surface with a hairline box, built-in padding, and an optional accent marker down the left edge.

  1. Surface — the flat fill (--color-bg); the card sits straight on the page, no shadow.
  2. Hairline box — a 1px rule on all four square sides (--color-rule-strong).
  3. Padding — built-in breathing room (--card-padding); removed by .card--flush.
  4. Accent marker — optional; a 3px left rule in ink or muted, flagging the panel like a changelog entry, via .card--accent-primary / .card--accent-secondary.

Variants

A base class plus optional, composable modifiers. The accent is a left-edge marker, not a colored box.

  • .card — the bordered surface, padded by default. Drop in any content.
  • .card--flush — removes the padding when the card is split into strips that pad themselves. The type specimens in the Typography section are flush cards.
  • .card--accent-primary — adds a 3px left marker in --color-ink.
  • .card--accent-secondary — adds a 3px left marker in --color-ink-muted, for a quieter flag.
  • .card--sunken — a recessed fill (--color-surface-sunken) for nested or read-only panels.

Default card

A padded surface with a hairline box. This is the default — reach for it first.

Primary marker

The same surface with a 3px ink marker down the left edge.

Secondary marker

A quieter, muted left marker for a parallel, lower-emphasis flag.

Sunken card

A recessed fill for nested or read-only panels, drawn from --color-surface-sunken.

Usage

Use a sectioning element (article / section) when the card is a content region; a plain div when it’s purely visual.

<article class="card flow">
<h3 class="type-h3">Card title</h3>
<p class="type-body-sm">Any content goes here.</p>
</article>

<!-- Ink left marker -->
<article class="card card--accent-primary">…</article>

<!-- Muted left marker -->
<article class="card card--accent-secondary">…</article>

<!-- Recessed surface for a nested panel -->
<article class="card card--sunken">…</article>

Accessibility

  • A card is a presentational surface — it adds no semantics. Put the real structure inside it (a heading, list, or landmark) so assistive tech still gets an outline.
  • Keep DOM order logical; the box groups content visually but doesn’t change reading order.
  • Don’t rely on the hairline alone to signal grouping — it uses --color-rule-strong (full ink) for exactly this reason. Keep it.
  • If a whole card is one target, wrap its content in a single <a> / <button> rather than nesting interactive elements, and keep a visible focus style.

Guidelines

Do

  • Reach for the default .card first; add a modifier only when the surface needs it.
  • Compose modifiers (.card--flush.card--accent-secondary) instead of writing a new card class.
  • Put real semantic structure inside; the card is only the surface.
  • Use .card--sunken for an inner panel instead of nesting full cards.

Don’t

  • Add drop shadows or faux elevation — the system stays flat; depth comes from the hairline box, not shadow.
  • Round the corners — the system is square everywhere (--radius-md is 0). Keep it consistent.
  • Nest bordered cards more than one level deep; the rules start to fight.
  • Use a card just to create space — that’s the job of .flow and the layout primitives.
  • Tint the surface or paint a full colored border; the accent is a left marker in ink or muted, nothing more.
